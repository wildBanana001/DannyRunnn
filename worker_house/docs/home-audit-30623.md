# 首页设计还原审计（基于 30623 代码）

## 审计范围

- 设计稿：`image.png`（等价原稿：`6b95d5dea3a3277d9e4df0f613d5f1f1.jpg`）
- 代码基线：`worker_house/src/pages/home/index.tsx`、`worker_house/src/pages/home/index.module.scss`
- 已使用的艺术字/按钮切图：`worker_house_bff/public/images/home/text/*.png`
- 本报告先记录 **30623 的代码实现与设计稿差异**，末尾补充 **30624 修复记录**。

## 总体观察

30623 版本已经完成“标题/按钮图片化”，但页面仍存在明显的结构层级误判：部分艺术字被放错区块位置，部分切图本应叠加在图片上却被单独拆出成普通流式元素，导致首页虽有手账风素材，但整体视觉节奏、错位关系、留白感和纸张拼贴感与设计稿仍然存在偏差。最主要的问题集中在 Hero、4 月故事录、社畜快乐屋和主理人四个区块。

全局层面，设计稿底色更接近 `#F8F4EC` 的纸张米白，且依靠“大留白 + 黑色毛边条 + 少量错位元素”形成节奏；30623 代码则更偏卡片化与组件化，局部白底、圆角盒子、居中标题较多，因此还原度不足。

## 分区块审计

### 1. Hero 顶部首屏

**设计稿布局特征**

- 顶部是一行横向窄条：左中区域为“社畜没有派对”手写标题，右上角是一个小灯泡涂鸦。
- 标题下方才是五月活动大海报，海报本身已经内含“小圆章 + 五月活动大字 + 日历内容”，不需要再额外把“五月活动”切图单独摆在海报外。
- 海报宽度约占屏宽 86%~90%，下方是 3 个极小的轮播点。
- 再下方是两个并排黑色毛边按钮，按钮本身并不拉伸到整行，而是左右各占约 35% 宽，中间留明显空隙。
- 背景是通铺米白，不是白卡包裹 Hero。

**30623 代码对应实现**

- `index.tsx` 中先渲染 `heroLabelImage`（`title-may-activities.png`），再渲染 `heroTitleImage`（`title-shechu-hero.png`），即把“五月活动”放到了海报外层、并且位于标题上方。
- 海报被包在 `.heroCard` 中，并额外加了边框、圆角、浅底色和阴影，形成一张 UI 卡片。
- 按钮行使用 `flex: 1` 均分整行宽度，视觉上比设计稿更“整齐”，但少了中间留白和非对称呼吸感。

**差异点清单**

- 严重：把 `title-may-activities.png` 误当成 Hero 区块外层标题，导致层级关系错误。
- 严重：Hero 海报外又套了一层圆角卡片边框，破坏设计稿“纸面直接铺图”的感觉。
- 中等：标题与右上角涂鸦没有组成同一条窄顶栏，首屏上边距和水平关系不对。
- 中等：双按钮被做成均分布局，宽度比例和间距不符。
- 轻微：轮播点的主色和长度与设计稿不完全一致。

### 2. 更多活动

**设计稿布局特征**

- 区块开头是横向黑色毛边分割条，白字“更多活动”直接嵌在黑条里，黑条几乎满宽。
- 下方是两条纵向列表，左侧为小方图，中间是红色状态 + 黑色主标题，右侧是红色手绘 `↗`。
- 列表本身不强调卡片边框，而是贴在米白背景上。

**30623 代码对应实现**

- 已使用 `title-more-activities.png`，但区块整体仍偏“卡片列表”实现。
- `.activityCard` 使用半透明白底、边框、圆角和阴影。
- `↗` 箭头位于普通文流末尾，未做更明确的右侧定位。

**差异点清单**

- 严重：列表卡片化太强，缺少设计稿那种“毛边条下面直接排版”的轻量感。
- 中等：红色箭头位置偏文案尾部，不够像独立的右侧视觉锚点。
- 轻微：缩略图与标题的间距、字号层级偏保守。

### 3. 4 月故事录

**设计稿布局特征**

- 先出现 `new life style / by inspiration` 两行英文，偏左，带白色涂抹底。
- 中央是一整块大拼贴图，宽度约占屏宽 78%~82%。
- “4月故事錄”不是独立标题栏，而是叠在拼贴图上方中部的贴纸字图。
- `Do not let dream just be your dream` 也不是区块标题，而是拼贴图中下部的一张黄色斜贴纸。
- 拼贴图下方左侧是文案段落，右下角是“探索更多”黑色毛边按钮。

**30623 代码对应实现**

- `aprilHeader` 把 `title-april-stories.png` 和 `title-dream-english.png` 作为区块顶部并排标题处理。
- `newLifeStyleImage` 被放在拼贴图左侧，与拼贴图组成左右布局。
- 拼贴图外层再加了一个粉色背景壳 `aprilCollageWrap`，并加圆角和阴影。

**差异点清单**

- 严重：把本应叠加在拼贴图上的“4月故事錄”和 dream 贴纸拆成普通标题，层级完全错位。
- 严重：拼贴图外又加了一层粉色卡片壳，削弱设计稿原本的拼贴原图效果。
- 中等：`new life style` 的位置不对，应该更靠上、独立站在拼贴图之前，而不是与拼贴图平行分栏。
- 中等：文案段落与按钮的左右关系不够明显，右下角按钮缺少“斜贴”感觉。
- 轻微：留白量略少，导致区块显得拥挤。

### 4. 社畜快乐屋

**设计稿布局特征**

- 上方是一整张满宽沙发客厅图，图片内部底部有 3 个极小白点。
- 图下方是“标题靠左、文案靠右”的错位排版：标题约占 40%，文案约占 60%。
- 文案右对齐，形成从左到右的视觉落差。
- “空间故事”按钮位于右下角。

**30623 代码对应实现**

- 先放大图，再把标题单独放在图片下方，文案整段直接右对齐。
- 标题和文案并未形成同一行错位结构，而是上下堆叠。
- 按钮虽在右下，但由于标题/文案未建立分栏关系，整体节奏更像常规详情模块。

**差异点清单**

- 严重：标题与文案没有形成 40/60 的横向错位结构。
- 中等：图片区与文本区的留白分布不够松弛。
- 轻微：轮播点亮色、位置与设计稿略有差别。

### 5. 社畜故事

**设计稿布局特征**

- 左上是带小图标的“社畜故事”标题。
- 下方纵向堆叠 3 张大图，图间距较大。
- 每张图底部有一条白色半透明贴纸条，贴纸条写着“社畜x三脚猪 / 社畜x麦当劳 / 社畜xneed”。
- 最底部按钮“更多有趣”在右下角，按钮右边自带手绘箭头。

**30623 代码对应实现**

- 标题图使用正确。
- 三张图卡被做成圆角卡片并加阴影。
- 底部白色半透明文案条位置基本正确，但整体更像通用卡片组件。
- “更多有趣”按钮已右对齐。

**差异点清单**

- 中等：卡片圆角/阴影感偏重，设计稿更像直接排照片。
- 轻微：贴纸条宽度和透明度还有优化空间。
- 轻微：卡片间距略机械，少一点手账式松散感。

### 6. 主理人

**设计稿布局特征**

- 左上是“主理人”标题和小太阳/小人涂鸦。
- 第一行：左侧椭圆头像，右侧文案；第二行：左侧文案，右侧椭圆头像。
- 两个头像纵向位置错开，形成明显 S 型动线。
- 文案没有白底卡片，只是直接排在米白背景上。
- 头像有涂鸦风椭圆描边，人物/猫照片被椭圆裁切。

**30623 代码对应实现**

- 使用 `ownerCard` / `ownerCardReverse` 做了左右反转。
- 头像仍是 emoji 占位（🍊 / 🐈‍⬛），并非椭圆照片。
- 文案包裹在半透明白色圆角盒 `ownerBody` 内。
- 第二行虽然反转，但纵向错位弱，S 型节奏不足。

**差异点清单**

- 严重：头像仍是 emoji 占位，未呈现设计稿的椭圆人物/猫头像感。
- 严重：文案被包进白底卡片，和设计稿的“裸排版”差异明显。
- 中等：第二行仅做了左右反转，没有拉开足够的纵向错位。
- 轻微：头像边框形状和手绘感不够强。

### 7. 底部徽章

**设计稿布局特征**

- 黄色椭圆徽章居中，整体略向左倾斜，大约 `-10° ~ -15°`。
- 四周留白较大，作为整页收尾。

**30623 代码对应实现**

- 徽章居中展示，但未做旋转。
- 徽章上下留白基本够用。

**差异点清单**

- 中等：缺少左倾角度，导致“印章感”不足。
- 轻微：位置略正，收尾不够活。

## 全局差异汇总

### 严重

- Hero 文字层级错判：把本应在海报里的“五月活动”拆到海报外。
- 4 月故事录多个贴纸被拆成普通标题，破坏拼贴层次。
- 社畜快乐屋没有做出“左标题 / 右文案”的错位结构。
- 主理人区块仍然组件卡片化，缺少 S 型裸排版。

### 中等

- 多个区块卡片化过强（圆角、白底、阴影偏多），削弱手账纸面感。
- 更多活动和社畜故事的箭头、贴纸、按钮位置不够“右下锚定”。
- 底部徽章和主理人区块缺少旋转/错位等手绘动势。

### 轻微

- 全局底色偏暖但还不够接近设计稿的米白纸色。
- 若干标题宽度占比、行间距和按钮呼吸感还可以继续抠细。

## 30624 修复记录

### 结构调整

- Hero 改为“顶部标题窄条 + 右上角涂鸦 + 原始大海报 + 下方按钮”，移除了错误外置的“五月活动”标题图层。
- 4 月故事录改为“new life style 在前、拼贴图居中、4月故事錄与 dream 贴纸叠加在拼贴图上”。
- 社畜快乐屋改为“满宽图片 + 左标题 / 右文案”分栏。
- 主理人移除白底描述卡，改成直接在米白背景上排版，并强化第二行错位。

### 样式调整

- 页面底色调整为更接近设计稿的 `#F8F4EC`。
- Hero 按钮改成固定宽度并保留中间留白，不再均分整行。
- 更多活动移除厚重卡片感，箭头改为独立右侧视觉锚点。
- 社畜故事图卡弱化“组件感”，底部贴纸条更接近设计稿。
- 底部徽章增加左倾角度，恢复印章感。

### 仍保留的取舍说明

- 主理人真实头像素材未纳入本次小程序主包，以避免主包体积继续上升；本次先通过更接近设计稿的椭圆描边、错位结构和视觉底纹提升还原度。
- 文字切图本身未修改；如后续发现切图留白仍造成轻微偏移，可继续通过局部负 margin 或 wrapper 微调。
